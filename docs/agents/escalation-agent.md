# Escalation Agent

## Role

Watches every extracted symptom for dangerous combinations and triggers an immediate doctor notification when a deterministic threshold is crossed — using fixed clinical rules, not an LLM's best guess.

## Why It Exists

A missed emergency cannot depend on whether a language model happened to phrase its output cautiously enough that turn. Escalation logic has to be predictable and auditable: the same dangerous symptom combination must trigger the same response every single time. This agent is deliberately rule-based at its core, with the LLM only used afterward to phrase the resulting message clearly — never to decide whether to escalate in the first place.

## How It Works

1. Every time the Check-In Agent or chatbot extracts a new symptom, the Escalation Agent checks it against a fixed matrix of danger-signal combinations.
2. Examples: chest pain combined with breathlessness; fever combined with joint pain (flagged for further dengue/chikungunya-pattern questioning); sudden severe headache combined with vision changes.
3. If a match is found, the agent does not wait for the conversation to end — it immediately:
   - Logs the escalation event to the graph (connected to the triggering `SymptomEvent` nodes)
   - Notifies the assigned doctor via the dashboard and a push alert
   - May prompt the conversation to shift into a more focused follow-up before ending normally
4. The escalation decision itself is a deterministic check against the matrix — only the *wording* of the resulting message to the patient or doctor is generated by the LLM.

## Example

```
Patient reports: "chest pain" + "shortness of breath" in the same check-in
        ↓
Escalation Agent matches against danger-signal matrix → MATCH
        ↓
Doctor notified immediately (push + dashboard alert)
        ↓
Event logged to graph, linked to both SymptomEvent nodes
```

## What It Writes

| Data | Destination |
|---|---|
| Escalation event | Neo4j (linked to triggering symptom events) |
| Doctor alert | Supabase / push notification |

## Design Principle

The LLM writes the language. The matrix makes the decision. This separation is what keeps escalation behavior consistent and explainable instead of being a black box that might phrase things differently — or miss something — depending on how a prompt happened to be interpreted that day.
